- [ ] **Mid-line HTML comments in `scanRegions`** (deferred from Phase 7 math-preview adversarial review, Session 15; confirmed 3/3 refuters). `core/qmd/model.ts` classifies HTML comments per WHOLE line (`COMMENT_OPEN`/`COMMENT_FULL_LINE`), so a comment that **opens mid-line** (`text <!--`) or **closes mid-line** (`--> $x$`) is mis-handled at the character level. Two symptoms in `core/math-regions.ts`: math inside a mid-line-opened comment is reported (false positive), and real math after a mid-line `-->` is missed (false negative). **This is a shared-model gap — it affects `core/refs.ts` (cross-ref index) identically**, not just math. A faithful fix needs character-level comment tracking in `scanRegions` (interacting with inline code spans + fences) and a full regression across outline/refs/citations — its own TDD pass (SAFEGUARDS: no cross-module refactor without plan mode). Low severity (mid-line comments are uncommon in `.qmd`). Documented in `core/math-regions.ts`'s docstring.

- [ ] **`findDiagramRegions` over-detects glued malformed cell info strings** (deferred from Phase 7 diagram-preview adversarial review, Session 16; confirmed 3/3 refuters, verified live against Quarto 1.7.33). `core/diagram-regions.ts` reports a diagram for `{mermaid=x}`, `{dot=1}`, `{mermaid#id}`, `{mermaid.foo}` — but Quarto renders **none** of these as a diagram (they become a plain code block with a `data-…` attribute, a literal `<pre>`, or inline `<code>`). Root cause is the **shared** `CELL_INFO` regex (`core/qmd/model.ts:138` — `/^\{([A-Za-z][A-Za-z0-9_-]*)[^}]*\}$/`) which captures the engine token greedily then accepts any trailing chars via `[^}]*`. **Shared-model gap — it affects every `findAllCells` consumer (outline / run-cell / math / refs)**, not just diagrams: those see `{mermaid=x}` as an executable cell too. A faithful fix tightens `CELL_INFO` to require the engine token to end at whitespace / comma / `}` while KEEPING the legitimate knitr comma forms (`{r,echo=FALSE}` renders) — cross-cutting, its own TDD pass with a full regression across all consumers (SAFEGUARDS: no cross-module refactor without plan mode). Low severity (only non-idiomatic / malformed info strings). Documented in `core/diagram-regions.ts`'s docstring.
