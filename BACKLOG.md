- [ ] **Mid-line HTML comments in `scanRegions`** (deferred from Phase 7 math-preview adversarial review, Session 15; confirmed 3/3 refuters). `core/qmd/model.ts` classifies HTML comments per WHOLE line (`COMMENT_OPEN`/`COMMENT_FULL_LINE`), so a comment that **opens mid-line** (`text <!--`) or **closes mid-line** (`--> $x$`) is mis-handled at the character level. Two symptoms in `core/math-regions.ts`: math inside a mid-line-opened comment is reported (false positive), and real math after a mid-line `-->` is missed (false negative). **This is a shared-model gap ‚Äî it affects `core/refs.ts` (cross-ref index) identically**, not just math. A faithful fix needs character-level comment tracking in `scanRegions` (interacting with inline code spans + fences) and a full regression across outline/refs/citations ‚Äî its own TDD pass (SAFEGUARDS: no cross-module refactor without plan mode). Low severity (mid-line comments are uncommon in `.qmd`). Documented in `core/math-regions.ts`'s docstring.
