# Roadmap

## Current Milestone
**v1 — the core authoring loop.** An installable, MIT-licensed `.vsix` that: highlights `.qmd`, renders, live-previews, runs code cells, and completes cross-references + citations — degrading gracefully when the Quarto CLI or a kernel is absent. Defined in `docs/planning/2026-06-27-extension-architecture-plan.md` §7. Architecture: TextMate grammar (Tier A) + in-process providers (Tier B), shelling out to the Quarto CLI. Visual editor / Zotero / Assist / project-wide indexing are out of v1. **Status (Session 12): all v1 features complete; the GitHub remote is wired (`origin` → `rmsharp/vscode_quarto_ext`, branch `master`); marketplace metadata + the listing README are done; and the F5 visual pass is done — 5 faithful screenshots (highlighting, outline, `@` completion, preview, render), captured in a dev host isolated with `--disable-extensions`, are wired into the README `## Screenshots` gallery (closing the "F5-only visual residue" every prior phase carried). The `npm audit` posture decision is done (Session 13): all 7 vulnerabilities are dev-only — none ship (`dependencies: {}`, no `node_modules` in the `.vsix`) — and are accepted + documented in `docs/SECURITY-AUDIT.md` (no `--force` fix, which would only break the toolchain). **v1 release prep is now agent-complete; the only remaining step before listing is the operator-only `vsce publish`** (Marketplace publisher `rmsharp` + PAT; `preview: true` is set).**

## What's Built
- **Phase 6c — Citation completion (Session 9). v1 FEATURE-COMPLETE.** Typing `@` in `.qmd` prose offers the citekeys from the document's `bibliography:` (BibTeX or CSL-JSON), with the entry title as detail, coexisting with cross-refs on `@` (VS Code merges the two providers and filters by typed text). Pure `core/frontmatter.ts` (`bibliographyPaths` — scalar/flow/block forms of the one key the region model skips) + `core/citations.ts` (`parseCitations`, plus `citationCompletionContext` — a citekey-aware `@`-token scanner that handles `:`/`.` keys); adapter `src/providers/citation.ts` reads the bib relative to the document and is gated to prose via `isReferenceableLine`. Verified by 8 real-host `executeCompletionItemProvider` tests (coexistence, in-cell gating, a colon-key whole-token replace, and three degradation paths) and hardened by a 5-lens / 3-verifier refute-by-default adversarial review (12 confirmed fixes — the key catch: reusing the cross-ref `ID_CHAR` scanner had silently broken colon/dot citekeys). **v1 — highlight / render / preview / run-cell / outline / cross-ref + citation completion — is feature-complete; next is release/packaging prep.**
- **Phase 6b — Cross-reference completion + go-to-definition (Session 8).** Typing `@` in `.qmd` prose offers the document's cross-ref labels (`@fig-/@tbl-/@sec-/@eq-/@lst-`) and Go to Definition jumps to a label's definition. Pure `core/refs.ts` indexes labels from three sources over the shared scanner (heading `{#sec-…}` ids — now kept as `Heading.id`; `#|`/`//|` `label:` cell options; inline `{#fig-…}` on prose), plus `isReferenceableLine`/`crossrefCompletionContext` helpers; adapter in `src/providers/crossref.ts` (completion trigger `@` + definition, both gated to prose). Verified by 9 real-host tests via `vscode.executeCompletionItemProvider`/`executeDefinitionProvider` (faithful, no CLI/Jupyter; incl. in-cell negative + prose control) and hardened by a 5-lens adversarial review (7 fixes; 1 indented-code-block phantom deferred). **v1 remaining: Phase 6c (citations) — the last v1 phase.**
- **Phase 6a — Document outline / symbols (Session 7).** A `DocumentSymbolProvider` populates the Outline view + breadcrumbs for `.qmd` with headings (nested by level) and code cells (leaves under their section). Establishes the shared **`core/qmd/model.ts`** region model — one `scanRegions` pass (front matter, HTML comments, fences, ATX headings) feeding `findHeadings`/`findAllCells`/`findCellAtPosition`/`buildOutline`; **Phase 5's `core/cells.ts` folded in** as a shim so cell-finding is shared with heading detection. Adapter in `src/providers/outline.ts`. Verified by a real-host test via `vscode.executeDocumentSymbolProvider` (faithful, no CLI/Jupyter) and hardened by a 5-lens adversarial review (6 fixes: front-matter/comment skip now shared with cells, 0–3-space fence-indent cap, `{#sec-id}` attribute stripped, empty closing-hash heading dropped). Setext headings deferred (backlog). **v1 remaining: Phase 6c.**
- **Phase 5 — `Quarto: Run Cell` family (Session 6).** `quarto.runCell` / `runCellAndAdvance` / `runCellsAbove` / `runAllCells` / `insertCell` find the code cell at the cursor and **delegate** execution to the user's Jupyter/R/Julia extension (feature-detected command IDs), degrading gracefully to a warning when none is installed. No CLI shell-out, no long-lived process; runs the in-editor buffer. `ctrl/shift+enter` bind only inside a cell (`quarto.inCodeCell` context key; `onLanguage:quarto` activation). Pure `core/cells.ts` (CommonMark backtick/tilde fence scanner, excludes plain/`{{}}`/`{.}`/nested fences) + `core/execution-delegate.ts`; adapter in `src/features/execution.ts`. Verified by 27 vitest + 8 real-host tests (faithful dispatch via a stand-in delegate) and hardened by an adversarial review (10 confirmed fixes). **v1 remaining: Phases 6a–6c.**
- **Phase 4 — `Quarto: Preview` (Session 5).** `quarto.preview` spawns `quarto preview --no-browser`, parses the `Browse at <url>` line (stderr, ANSI-wrapped), and embeds the live preview in a webview beside the editor (auto-reload on save via Quarto livereload). Owns the process lifecycle — the server **and its detached deno worker** are reaped (process-group kill) on pane close / doc close / deactivate, with no orphan. Pure `core/preview-url.ts` + `core/preview-html.ts` (CSP-locked); adapter in `src/features/preview.ts`. Verified by a real-host integration test that asserts the worker is reaped (faithful no-orphan probe), hardened by an adversarial review (TOCTOU race + 4 fixes).
- **Phase 3 — `Quarto: Render` (Session 4).** The `quarto.render` command shells `quarto render <active.qmd>`, streams output to a dedicated Output channel, shows the produced path on success (with an Open action) or surfaces the error verbatim on non-zero exit, and fails soft when the CLI is absent. Pure `core/render-args.ts` (arg-builder + ANSI-tolerant output-path parser) keeps logic headlessly testable. Verified: success renders to HTML and a deterministic-failure fixture confirms no crash, both in a real `@vscode/test-electron` host. Phases 4–6c remain.
- **Phase 2 — `.qmd` highlighting (Session 3).** `.qmd`/`.rmd` register as the `quarto` language; an original `text.html.quarto` grammar highlights prose (via the built-in markdown grammar by reference), YAML front matter, and `{python}/{r}/{julia}/{ojs}` cells, with bracket/comment/folding config. Verified headlessly via `vscode-textmate` tokenization + a real-host registration test.
- **Phase 1 — walking skeleton (Session 2).** Installable `.vsix` builds from a TypeScript/esbuild scaffold; the `core/`-vs-adapter boundary is established (`vscode`-free `core/`); the `Quarto: Verify Installation` command resolves the CLI and reports its version; unit (vitest) + integration (`@vscode/test-electron`) test harnesses both run green headlessly.
